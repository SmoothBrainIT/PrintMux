# Contributing to PrintMux

Thanks for considering a contribution!

## How to Contribute
1. Fork the repo and create your branch from `main`.
2. Keep changes focused and well-scoped.
3. Add or update tests when appropriate.
4. Open a pull request with clear context and screenshots for UI changes.

## Development Setup
See `README.md` for local setup instructions.

## Code Style
- Keep changes readable and consistent with existing patterns.
- Prefer explicit, descriptive names.
- Avoid large refactors in the same PR as feature work.

## Commit Messages
Use short, descriptive commit messages that explain *why* the change was made.

## Reporting Bugs
Please include:
- Steps to reproduce
- Expected vs actual behavior
- Logs or screenshots if available

## Security Issues
For security-related issues, please see `SECURITY.md`.
